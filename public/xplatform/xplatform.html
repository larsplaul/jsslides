Cross Platform Development

Judging by the number of available technologies, this seems like the way many has chosen.
The following are just af few examples

  • Xamarin → Write code in C#. Tightly bound to C# and Visual Studio and not completely free
  • Codename One → Write code in Java. A set of Software Development Tools aiming to provide Write Once, Run Anywhere code for various desktop and mobile operating systems like Windows macOS Android and iOS
  • Appcelerator Titanium → Open-source framework that allows the creation of mobile apps on platforms including iOS, Android and Windows Phone from a single JavaScript codebase
  • Apache Cordova (Phonegap) → Framework that enables developers to build mobile apps using CSS3, HTML5, and JavaScript instead of relying on platform-specific APIs
  • Intels Multi-OS Engine → New plug-in technology that integrates into Android Studio. Developers can start a mobile app project in Android Studio using Java, then utilize the Multi-OS engine to configure the project to build and run as an iOS* App on the iOS* simulator

Which ones should we go for?

Since this course is titled "Full Stack JavaScript" we will focus on JavaScript technologies from here on, but don't forget: there are many alternatives

There seems to be a general acceptance (especially if you don't read articles tied to a specific technology) that Native always gives the best user experience

  • Native → High User Experience
    With "JavaScript" this is only possible using the newest technologies like NativeScript and React Native (next slide)
    Real Native is probably the way all would go, if "cost was not an issue" or using strategies like above if/when they mature.
  • Hybrid → Lower User Experience
    Don't read lower as "not acceptable". When development costs are taken into account this is a strategy accepted by many companies.
    This is the strategy used by Cordova and Ionic (which uses Cordova), and what we are going to use in this course
  • WEB → Low User Experience This is either pure responsive web pages users have navigated to with their browser, or responsive pages running in an embedded web-view. Will never get the true look of a native app

Some of the JavaScript "newcomers"

NativeScript

An open source framework for building truly native mobile apps with JavaScript. Use web skills, like Angular, FlexBox and CSS, and get native UI and performance on iOS and Android.

Telerik (the company behind NativeScript) and the Angular team works closely together so Angular 2 is the suggested JS framework to use

React Native

Like with NativeScript this is a technology where you build a real mobile app that's indistinguishable from an app built using Objective-C or Java.

The building blocks used is JavaScript and React. The fact that Facebook uses React Native serves as the ultimate "proof of concept"

The main difference between these two technologies are:

NativeScript → This is more a "write once use anywhere" solution where Angular 2 skills can be used for both Web and App development

React Native → Facebook refer to their strategy as "Learn once, use anywhere". React and React Native is not the same, but as a developer your are using the same overall strategy
