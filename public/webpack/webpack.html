What is Module Bundling and why bother?

On a high level, module bundling is simply the process of stitching together a group of modules (and their dependencies) into a single file(or group of files) in the correct order.

Why Bundle files?

When you divide your program into modules, you typically organize those modules into different files and folders (think React Components).
As a result, each of those files has to be included in your main HTML file in a <script> tag, which is then loaded by the browser when a user visits your home page.
Having separate <script> tags for each file means that the browser has to load each file individually:
→This has a cost in terms of page load time.

To get around this problem, we bundle all files into one big file (or a couple files) in order to reduce the number of requests.
Another common approach to speed up the bundling operation is to “minify” the bundled code.
Minification is the process of removing unnecessary characters from source code (those only meant for humans).

Ways to Bundle

Non Standard

CommonJS

CommonJS is a project to define a common API and ecosystem for JavaScript.
One part of CommonJS is the Module specification. Node.js is a server-side JavaScript runtimes that implement modules based on the CommonJS Module specification

Asynchronous module definition (AMD)

AMD is another specification for modules. RequireJS is probably the most popular implementation of AMD. One major difference from CommonJS is that AMD specifies that modules are loaded asynchronously.

Native JS (es2015)

ES6 Modules

As we have seen, es2015 and forward has native support for Modules.

WebPack - a (very) quick intro

Install it
Global: npm i -g webpack
Local : npm i webpack --save-d

First Run:
webpack <entrypoint> <output>

webpack main.js bundle.js

Defining a Config File

webpack uses a principle of an entry point and an output location, just like int the command above

module.exports = {
  entry: './main.js',
  output: {
             filename: './bundle.js'
          }
};

Configuring Watch Mode

Signal webpack to watch for the code for change and automatically "rerun" the webpack command. 

module.exports = {
  entry: './main.js',
  watch; true,
  ...
};

Loaders

Loaders are transformations that are applied on a resource file of your app.
They are functions that take the source of a resource file as the parameter and return the new source. For example, we can use loaders to tell webpack to load:
  • ES-6 and transpile with Babel
  • TypeScript and transpile with tsc
  • ...
  • test A Regular expression that tests what kind of files to run through this loader.
  • exclude  Which files the loader should exclude /ignore.
  • loader The name of the loader we are going to use (babel-loader)
  • query   Here we can pass options to the loader
  • presets  Use the es2015 presets that were installed earlier (babel)
module.exports = {
  // ...
  module: {
    loaders: [
        {
          test: /\.js$/,
          exclude: /node_modules/,
          loader: 'babel-loader',
          query: {
                 presets: ['es2015']
                }
        }
     ],...
