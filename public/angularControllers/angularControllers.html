Angular - Controllers

In Angular, a Controller is a JavaScript constructor function that is used to augment the Angular Scope.

Before we go into Controllers we need to understand Scopes

Controllers

$Scope

A scope can be seen as the glue which allows the template, model and controller to work together. Angular uses scopes, along with the information contained in the template, data model, and controller, to keep models and views separate, but in sync. Any changes made to the model are reflected in the view; any changes that occur in the view are reflected in the model.

Scopes

Scope serve as the glue between the controller and the view

//Controller
angular.module('scopeExample', []);
  .controller('MyCtrl', ['$scope',
    function($scope){
      $scope.name = 'World';

      $scope.sayHello = function() {
      $scope.greeting = 'Hello '
      + $scope.name;
    };
}]);
<!-- View -->
<div ng-controller="MyCtrl">
  Your name:
  <input type="text" ng-model="name">
  <button ng-click='sayHello()'>greet</button>
  <hr>
  {{ greeting }}
</div>
Your name:
{{greeting}}

$scope Versus controllerAs Syntax

In the previous example we injected the $scope variable into the controller.
In AngularJS 1.2 and later, there is a new syntax, the controllerAs syntax, which allows us to define the variables on the controller instance using the this keyword, and refer to them through the controller from the HTML. 
//Controller
angular.module('scopeExample', []);
 .controller('MyCtrl2', [function(){
   var self = this;
   self.name = 'World';
   self.sayHello = function () {
   self.greeting = 'Hello ' + self.name;
 };
}]);
<!-- View -->
<div ng-controller="MyCtrl2 as ctrl">
  Your name:
  <input type="text" ng-model="ctrl.name">
  <button ng-click='ctrl.sayHello()'>ctrl.greet</button>
  <hr>
  {{ ctrl.greeting }}
</div>
Your name:
{{ctrl.greeting}}

Understanding Controllers

When a Controller is attached to the DOM via the ng-controller directive, Angular will instantiate a new Controller object, using the specified Controller's constructor function. A new child scope will be available as an injectable parameter to the Controller's constructor function as $scope.

Use controllers to:

  • Set up the initial state of the $scope object. (first line in controller)
  • Add behavior to the $scope object. (via ng-click)

ng-model - Binding to input controls

Important directives when binding between Templates and Controllers

ng-controller
The ng-controller directive attaches a controller class to the view
ng-model
The ng-model directive binds an input, select, or textarea (or custom form control) to a property on the scope

Two Way Bindings

We have a two way data binding when a model variable is bound to a HTML element that can both change and display the value of the variable. In general, we could have more than one HTML element bound to the same variable

We use the ng-model directive to bind a model variable to a HTML element that can both display and change a value.

Lifecycle of Angular Controllers

Controllers are not singletons. Anyone can create a new controller and they are never auto-destroyed.
The fact is that it's generally bound to the life cycle of its underlying scope. The controller is not automatically destroyed whenever its scope is destroyed.
However, after destroying an underlying scope, its controller is useless.

If you wan't singleton behaviour, refactor code into an Angular Service and inject this service into your controller.

Your controller could be instantiated many times, but all instances will get the same single instance of the service
